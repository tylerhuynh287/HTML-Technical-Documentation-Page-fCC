Introduction

HTML (HyperText Markup Language) is the backbone of the web. It structures content such as text, images, links, and multimedia on web pages. HTML tells browsers how to display content, but not how it should look â€” that's where CSS and JavaScript come in.

First introduced by Tim Berners-Lee in 1991, HTML has gone through multiple revisions. The latest version, HTML5, introduced semantic tags and native support for media and form validation.

HTML is not case-sensitive, but conventionally tags are written in lowercase for consistency.

Syntax

HTML uses elements made of tags enclosed in angle brackets. Most elements follow an opening/closing tag pattern:

<p>This is a paragraph</p>

Elements can be nested, and indentation helps readability. Some elements are self-closing:

<br> <img src="image.jpg" alt="Description">

Attributes provide additional information:

<a href="https://example.com" target="_blank">Visit Site</a>

Always quote attribute values and use meaningful alt text for images to improve accessibility.

Advanced Elements

HTML5 introduced semantic elements for better structure:

  • <header>, <footer>: Introduce top/bottom content areas
  • <main>, <section>, <article>: Define main content and its parts
  • <aside>: Side content or supplementary info
  • <nav>: Groups site navigation links

These tags enhance accessibility and SEO. They also help developers and tools (like screen readers) understand the layout.

<main>
  <article>
    <header><h1>Title</h1></header>
    <p>Body content here...</p>
    <footer>Author name</footer>
  </article>
</main>
Forms

Forms collect user input and are essential for interactive websites. Basic form elements include:

  • <input type="text">, <input type="email">, <input type="password">
  • <textarea>: For longer messages
  • <select> and <option>: Dropdown menus
  • <button>: For form submission or actions

Example form with basic validation:

<form action="/submit" method="post">
  <label for="name">Name:</label>
  <input id="name" name="name" required><br>
  <label for="email">Email:</label>
  <input id="email" name="email" type="email" required><br>
  <button type="submit">Submit</button>
</form>

Use label elements to link inputs for accessibility and screen readers.

Inline vs Block Elements

HTML elements are either block-level or inline:

  • Block-level: Start on a new line, take up full width. Ex: <div>, <p>, <h1>
  • Inline: Stay in flow with surrounding content. Ex: <span>, <a>, <strong>
<p>This is <span>inline</span> content inside a paragraph.</p>

Understanding this is key for styling layouts with CSS.

Media & Embeds

HTML allows embedding various media types directly into web pages:

  • <img>: Embeds an image. Use alt for accessibility.
  • <video>: Embeds a video with controls.
  • <audio>: Embeds sound clips or music.
  • <iframe>: Embeds another webpage or a YouTube video.
<video controls width="300">
  <source src="video.mp4" type="video/mp4">
  Your browser does not support the video tag.
</video>

For YouTube embeds, use:

<iframe width="560" height="315" src="https://www.youtube.com/embed/example" frameborder="0" allowfullscreen></iframe>
Best Practices & SEO

Writing clean, accessible, and semantic HTML improves usability and visibility in search engines.

  • Use semantic tags wherever possible (e.g. <article>, <main>)
  • Always use alt attributes on images
  • Structure content using headings in order: <h1> to <h6>
  • Use aria-* attributes for accessibility enhancements when needed
  • Minimize inline styles and prefer external CSS for maintainability

Proper HTML structure makes your website more performant, accessible, and easier to maintain.
